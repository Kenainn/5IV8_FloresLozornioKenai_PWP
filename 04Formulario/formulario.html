Todos los formularios en la web manejan métodos para el envío de las diferentes peticiones que realiza el usuario, considerando la arquitectura cliente-servidor. El manejo de las peticiones se hace por medio del protocolo HTTP y actualmente HTTPS. Cada petición debe ser atendida en un tiempo de respuesta conforme al modelo TCP/IP. Los métodos principales de los formularios son:

  1. GET: Este método se encarga de enviar la información del formulario por medio de la URL del navegador. Esto puede permitir inyecciones de código y violaciones de seguridad.
  2. POST: Este método se encarga de enviar la información para registrar cambios o nuevos datos en el sistema. Los datos no son visibles en la URL, lo que ofrece mayor seguridad.

En resumen, los métodos son:

  • GET: Sirve para enviar y realizar operaciones de lectura o consulta.
  • POST: Sirve para enviar información a un recurso en el servidor con el fin de registrarlo.
  • DELETE: Sirve para eliminar un recurso específico dentro del servidor obteniendo una respuesta de actualización.
  • PUT: Sirve para almacenar la información de un recurso particular y conectarse al servicio para actualizarlo.
  • HEAD: Sirve para regresar las cabeceras HTTP asociadas a un recurso.
